# Valuereporter

## Purpose
Server application that collects and visualizes service usage statistics. Receives observations from Valuereporter-Agent and Valuereporter-Java-Client about which classes and methods are being used in production, then presents usage statistics through an intuitive GUI. Helps identify unused code and measure the value generated by each service.

## Tech Stack
- Language: Java 21
- Framework: Embedded web server with GUI
- Build: Maven
- Key dependencies: Valuereporter-Agent, Flyway (database migrations), SLF4J
- Persistence: Embedded database (default) or external database

## Architecture
Standalone server application with embedded database and web GUI. Receives usage telemetry data via HTTP from Valuereporter-Agent instances running alongside production applications. Stores observations in a relational database (managed by Flyway migrations) and presents aggregated statistics through a web interface showing functionality usage percentages.

## Key Entry Points
- `config_override/valuereporter.properties` - Configuration overrides
- `resources/db/initialize_new_database.sql` - Database initialization
- Main JAR: `java -jar target/valuereporter-*-with-deps.jar`

## Development
```bash
# Build
mvn clean install

# Run
java -jar target/valuereporter-*-with-deps.jar

# Configure
cp src/main/resources/valuereporter.properties config_override/
```

## Domain Context
Production observability and code value analysis. Tracks which application functionality is actually being used in production, enabling teams to identify dead code, measure feature adoption, and understand service value.
